## 二 、文章假定
- **假说1（基准效应）**： 数字贸易规则的深化通过降低双边贸易成本与制度摩擦，显著提升本国数字服务出口网络韧性
- **假说2（机制路径：稳定器与竞争效应）**：参考国际贸易网络演化理论，数字贸易规则敞口通过“扩大进口规模”、“提升制度环境水平”以及“平抑进口需求波动”发挥稳定器作用；但同时也会加剧市场竞争，可能存在非线性的边际效用递减。
- **假说3（结构异质性与马太效应）**： 在全球“核心-边缘”网络结构中，核心国家凭借过饱和的连接存在“锦上添花”的边际递减，而边缘国家搭靠核心节点能实现从0到1的枢纽跃升；但从数字规则制定的红利吸收看，发达国家与发展中国家之间可能存在强者恒强的“马太效应
- **假说4（第三方效应溢出）**： 一国数字贸易规则网络地位的提升不仅直接作用于缔约国，还能通过网络桥接效应（Structural Holes）和规则外溢，对未缔约的第三方国家产生正向的出口网络韧性拉动

## 变量选取与模型设定

### 4.1 被解释变量：真实网络韧性 (True Resilience)
本文的被解释变量为**数字服务出口网络真实韧性（$True\_Resilience_{it}$）**。与现有文献简单使用“波动项与趋势项比值”不同，本文在概念与数理逻辑上进行了双重重构，以准确刻画一国在面对外部冲击时的贸易网络拓扑稳定性。具体构建步骤如下：

1. **测算网络拓扑广度**：基于 BaTiS 双边服务贸易数据，采用年度中位数截断法（$\tau_t = \text{Median}(V_{ijt})$）构建去噪后的无权有向网络，计算出国家 $i$ 在 $t$ 年的标准化出度中心度（$C_{it}$），以此反映其数字服务出口市场的辐射广度。
2. **HP 滤波时序分解**：运用 Hodrick-Prescott 滤波（$\lambda=6.25$）将出度中心度时间序列分解为长期趋势项（$\tau_{it}$）和短期波动项（$c_{it}$），提取绝对波动率 $V_{it} = |c_{it}|$。
3. **逻辑纠偏与极值保护**：基础的变异系数（$V_{it} / \tau_{it}$）衡量的是系统的“脆性”。为了防止核心枢纽国家趋势项接近 0 导致的数值爆炸溢出，本文引入极小值截断底板，并对该比值取自然对数后**取相反数**。最终公式为：
   $$True\_Resilience_{it} = - \ln\left(\frac{V_{it}}{\max(\tau_{it}, 0.01)} + 10^{-6}\right)$$
   该指标数值越大，表明一国网络拓扑结构的波动性越低、抗冲击能力越强，即网络韧性越高。

### 4.2 核心解释变量：数字贸易规则政策敞口 (Exposure)
本文的核心解释变量为**连续型政策敞口指数（$Exposure_{it}$）**。为克服传统节点级虚拟变量（0/1）强制降维导致的双边网络拓扑信息丢失及衰减偏误，本文将数字贸易规则（TAPED）从“边”属性有效聚合至“节点”层面。

公式定义如下：
$$Exposure_{it} = \sum_{j \neq i} \left( w_{ij} \times TAPED_{ijt} \right)$$
其中，$TAPED_{ijt}$ 表示 $t$ 年国家 $i$ 和国家 $j$ 之间是否实施了数字贸易规则（或其条款深度）。$w_{ij}$ 为**静态基期贸易权重**，计算公式为：
$$w_{ij} = \frac{Trade_{ij, t_0}}{\sum_{j \neq i} Trade_{ij, t_0}}$$
本文选取数字贸易规则爆发前的 2005-2007 年作为静态基期（$t_0$）计算 $w_{ij}$。该设定不仅测度了本国面临数字贸易规则冲击的真实“敞口浓度”，还利用“基期快照”切断了“签订规则与当年贸易额激增”之间的逆向因果路径，有效缓解了内生性问题。

### 4.3 控制变量 (Control Variables)
为控制可能同时影响数字贸易规则签订和出口网络韧性的宏观因素，借鉴世界银行（World Bank）等权威数据库，本文在模型中引入以下随时间变化的国家特征变量集（$X_{it}$）：
1. **经济发展水平**：人均 GDP 的对数（ln_GDPer），反映一国整体经济实力。
2. **经济增长潜力**：GDP 增长率（GDP_Growth），反映宏观经济的动态活力。
3. **外资吸引力**：外商直接投资净流入占 GDP 的比重（FDI_GDP），控制跨国资本流动对贸易网络的溢出效应。
4. **数字化基础设施**：互联网普及率（Internet_Usage，即使用互联网人口百分比），是发展数字贸易的核心硬件先决条件。
5. **制度环境质量**：基于全球治理指标（WGI）提取的法治水平或综合治理指数（WGI_RuleOfLaw），控制东道国契约执行效力对贸易信任网络的支撑作用。

### 4.4 经验模型设定：多时点双向固定效应模型 (TWFE DID)
在上述指标体系的基础上，为了因果识别数字贸易规则网络渗透度对数字服务出口网络韧性的影响，本文构建如下多时点双向固定效应（TWFE）双重差分模型：

$$True\_Resilience_{it} = \alpha + \beta Exposure_{it} + \gamma X_{it} + \mu_i + \lambda_t + \varepsilon_{it}$$

* **参数解释**：
  * $True\_Resilience_{it}$ 为核心被解释变量。
  * $Exposure_{it}$ 为核心解释变量（处理变量）。
  * $\beta$ 为本文关注的核心估计参数（平均处理效应 ATT）。如果 $\beta$ 显著大于 0，则表明随着与贸易伙伴签署数字贸易规则的敞口加深，该国的出口网络韧性得到了实质性提升。
  * $X_{it}$ 为一系列控制变量的集合，$\gamma$ 为其系数向量。
  * $\mu_i$ 为国家固定效应，用于吸收地理位置、文化渊源、历史语言等不随时间改变的国家异质性特征。
  * $\lambda_t$ 为年份固定效应，用于控制诸如 2008 年金融危机、新冠疫情等对全球所有国家造成共同冲击的宏观时间趋势。
  * $\varepsilon_{it}$ 为随机扰动项（通常在国家层面上进行聚类调整，即 Cluster Robust Standard Errors，以处理面板数据潜在的自相关和异方差问题）。
# 前言
## 1.对标论文的缺陷
### 自变量维度的“暴力降维”（双边网络属性被抹杀）
TAPED 数据库记录的是双边或区域贸易协定（PTAs/RTAs），也就是说，规则是建立在“边（Edge）”上的 。比如，中国和东盟签署了涵盖数据流动的协定。
但是，看论文的模型设定式(4)：该文将 $Rules_{i,t}$ 定义为节点级别（Node-level）的虚拟变量——只要一国在 $t$ 年签订了数字贸易规则，该变量就取 1 。
致命逻辑漏洞： 如果中国仅与东盟签署了协定，这篇论文的处理方式会使得中国在当年面对美国、欧洲等所有未签协定国家时，其 $Rules$ 变量也变成了 1。这种将“双边连线”强行压缩为“单边节点属性”的做法，彻底稀释了政策的真实冲击力度，引入了巨大的测量误差。你的 P 值跑不出来，很大程度是因为真实的双边政策效应被这种单边虚拟变量的设定给“淹没”了。
下面我们展开来讲这个问题：

我们设 $R_{ijt}$ 为 $t$ 时期国家 $i$ 和国家 $j$ 之间的数字贸易规则深度（可以是一个连续的分数，也可以是 0 或 1）。政策的冲击是精准落在 $i$ 和 $j$ 之间的那条 **“边”** 上的。该论文（以及可能之前的你）强行把这个双边变量压缩成了单边变量 $Rules_{it}$。其数学逻辑实质上是：
$$Rules_{it} = \begin{cases} 1, & \text{if } \sum_{j} R_{ijt} > 0 \\ 0, & \text{if } \sum_{j} R_{ijt} = 0 \end{cases}$$

假设中国（$i$）在 2020 年仅与新加坡（$j$）签署了深度数字贸易协定（$R_{C,S} = 1$），但与美国、欧盟等最大的贸易伙伴（$k$）都没有签署（$R_{C,k} = 0$）。如果你把中国的节点变量强制设为 $Rules_{C, 2020} = 1$，那么在回归方程中，模型会认为中国在 2020 年数字贸易出口的整体变化都受到了政策的干预。但实际上，中国对美、欧的庞大出口网络并没有享受到政策红利。模型在拟合时，看到的是：“哎？中国明明受到了政策干预（$Rules=1$），怎么它对美欧的贸易连线却没有明显变强？”此时，巨大的未受干预样本（噪音）稀释了极小受干预样本（信号）的真实效应。在计量经济学中，自变量存在严重的测量误差（Measurement Error）时，会导致估计系数 $\beta$ 发生衰减偏误（Attenuation Bias）——也就是估计值会无可挽回地向 0 靠拢。
### 严重违背多时点 DID 的 SUTVA 假设（这个因为本身不可能完全符合，所以不展开）
### 挂羊头卖狗肉的“伪”网络韧性
根据之前的学习，度量网络韧性有一个相对简单的办法，那就是在随机或蓄意攻击下，最大连通分量（LCC）的缩减比例。
但是，看该论文的式(3)：$Express = ExportDegree_{volatility} / ExportDegree_{trend}$ 。
致命逻辑漏洞： 作者仅仅是把每个国家（节点）自身的“出度中心度”做了一个 HP 滤波，求出波动项和趋势项的比值 。这在本质上只是衡量了单一节点出口规模的波动率（出口稳定性），完全没有触及网络拓扑结构在面对节点失效时的抗崩溃能力。它测算的是“节点的稳定性”，而非“网络的韧性”。

### 2.数据库的缺陷：
BaTIS 为了提供一个“完全平衡且无缺失”的矩阵，大量使用了 **结构引力模型（Gravity Model）** 去填补（Impute）那些官方没有报告的、或者极小的缺失贸易流。

# 第一节：数据检查和清洗
## 1.1 数据库的选取和理由
考虑下列因素，我们决定应用BaTiS进行因变量即网络韧性的构建。理由如下
1. 网络拓扑的完整性（205 节点 vs 99 节点）：网络科学对节点的完备性要求极高。TiVA 的 99 个节点主要局限于 OECD 及主要新兴经济体，这在拓扑学上是一个被“人为截断的富人俱乐部”。数字贸易规则（TAPED）的异质性往往体现在边缘发展中国家因合规成本过高而被“踢出”网络。BaTIS 包含 205 个独立经济体，能够完美还原全球贸易网络真实的“核心-边缘（Core-Periphery）”结构，这是测算全局韧性（LCC）的根本前提。
2. 经济学传导机制的直接性（总贸易 vs 附加值）：BaTIS 记录的是双边总贸易额（Gross Trade），而 TiVA 是附加值（Value-Added）。TAPED 中的数字协定条款（如禁止强制源代码审查、跨境数据流动便利化）直接削减的是“双边名义贸易成本（冰山成本）”。政策冲击最先、最直接反映在双边总贸易流上。如果用附加值去构建“基期贸易权重”，多国嵌套生产的折算会严重稀释双边政策的真实映射。
3. 时间窗口的覆盖度（至2024 vs 至2021）：BaTIS 更新至 2024 年，完整覆盖了后疫情时代数字经济爆发以及全球（如 DEPA 等）数字规则密集签署的关键期。
   
此外，处于审慎性考虑，我们在一开始的代码中便剔除了像欧盟和WLD（全球）这样的经济聚合体

## 1.2 权重矩阵的生成与可视化
### 1.2.1 权重矩阵的必要性
1. **修正“暴力降维”的致命测量误差：**
原论文把 TAPED 数据库中“国与国之间（双边）”的数字贸易协定，简单粗暴地压缩成了“只要签了就是 1”的单边虚拟变量。这稀释了政策真实的冲击力度，导致旧有的回归跑不出显著的 P 值。
2. **构建“连续敞口指数（Exposure Index）”：**
为了在不改变基础多期 DID 模型的前提下解决上述问题，我们需要把自变量变成一个连续的强度指标。核心公式是：$$Exposure_{it} = \sum_{j \neq i} \left( w_{ij} \times TAPED_{ijt} \right)$$其中 $w_{ij}$ 就是我们要计算的权重。如果本国 $i$ 跟贸易伙伴 $j$ 的业务占比很大（$w_{ij}$ 很高），那么一旦签订协定（$TAPED=1$），受到的政策冲击（敞口）就大。
3. **使用“静态基期”防御内生性（逆向因果）：**
我们在代码中挑出了base_years=[2005, 2006, 2007]。因为如果是用“当期”的贸易额做权重，就会产生内生性循环：
<div style="text-align: center; background-color: #f8f9fa; padding: 15px; border-left: 5px solid #007bff; margin: 10px 0;">

**内生性循环逻辑：**
签订贸易协定 $\implies$ 贸易额激增 $\implies$ 当期权重 $w_{ij} \uparrow$ $\implies$ 高估 $Exposure_{it}$
</div>
这样你根本分不清是权重导致了协定，还是协定导致了权重。 

**而使用 2005-2007 年的静态基期（政策大面积发生前），相当于给全球贸易结构拍了一张“快照”并彻底冻结，从而斩断了未来的内生性干扰**  

### 1.2.2 权重矩阵的生成方式
在代码中，我们设计了`build_batis_static_weights`这个函数。其执行步骤如下
1. 截取基期并平滑波动:。代码提取了 2005-2007 年的数据，并用 .mean() 求平均值 $Mean(Trade_{ij})$，计算出 $i$ 国到 $j$ 国最稳定、最真实的贸易底色。
``` python
df_base = clean_df[clean_df[time_col].isin(base_years)]
trade_ij = df_base.groupby([ref_area_col, counterpart_col])[value_col].mean()
```
2. 删除自循环：在国际贸易网络中，本国对自己国内的服务销售不属于“跨国贸易”。如果不剔除 $i=j$ 的情况，会导致网络拓扑学上的“自连接”，破坏后续的网络距离计算。
``` python
trade_ij = trade_ij[trade_ij[ref_area_col] != trade_ij[counterpart_col]]
```
3. 计算全球总出口额（分母）:计算 $i$ 国在基期内对全世界所有其他国家的出口总额：$\sum_{j \neq i} Trade_{ij, t_0}$。注意，**代码用的是“数据框内已知国家的汇总加总”，而不是外部的宏观数据（WLD），这确保了分母和分子在同一个封闭的微观样本池内。**
``` python
trade_i_total = trade_ij.groupby(ref_area_col)['Base_Trade_ij'].sum()
```

4. 执行核心代数运算：$$w_{ij} = \frac{Trade_{ij, t_0}}{\sum_{j \neq i} Trade_{ij, t_0}}$$这确保了对于任意一个出口国 $i$，其对所有伙伴国 $j$ 的权重之和绝对等于 1（即 $\sum w_{ij} = 1$
``` python
weight_matrix['w_ij'] = weight_matrix['Base_Trade_ij'] / weight_matrix['Base_Trade_i_Total']
```

对于深红色区域的“核心国家”，它们的网络连接已经过饱和，签订一个新协定对其网络韧性的边际提升微乎其微；但对于浅黄色区域的“边缘国家”，一旦通过数字协定与某个大国建立了一条新的连线，其在网络中的“枢纽性（中心度）”就会发生从 0 到 1 的质变。这就是为什么异质性检验中，边缘国家的 P 值反而极其显著的物理根源。

#### P.S. 什么是异质性分析？
“网络位置异质性”，就是专门研究：国家在全球贸易网络中处于不同的地位（核心大国 vs 边缘小国），签订数字贸易规则对其网络韧性的提升效果是否会有差异？
- 回忆一下你刚画的热力图。美国、德国（处于深红色核心区）的网络连线已经密密麻麻，处于“过饱和”状态，抗风险能力本来就强。它们多签一个协议，可能只是锦上添花，边际效应递减 。
- 但对于阿鲁巴、塔吉克斯坦（处于浅黄色边缘区）来说，它们原本在网络中孤立无援。如果它们通过数字贸易规则，成功抱上了一个核心大国的大腿，建立了一条粗壮的连线，它们在网络中的枢纽地位就会发生“从 0 到 1”的质变，韧性大幅飙升 。
- **这个检验就是为了用数据在数学上证明上述直觉：规则对边缘国家的“雪中送炭”作用，远大于对核心国家的“锦上添花”作用 。**

##### 操作步骤
1. 指标排序：把你算出来的各国每年的网络中心度（比如我们刚算出来的Out_Degree_Centrality 或 Degree）拿出来，按照数值从小到大排序
2. 切割分组：根据分位数（比如 25%、50%、75%），把所有国家切分成 4 个平行宇宙（四组）：
   1. 最边缘国家（0 - 25%）
   2. 次边缘国家（25% - 50%）
   3. 次中心国家（50% - 75%）
   4. 中心国家（75% - 100%）
3. 分别跑回归：用你完全相同的多期 DID 计量模型，分别在这 4 个切分出来的小样本数据集上各自跑一次回归
4. 对比结果（P值与系数）：
观察哪几组的星号亮了（显著）。在原论文中，结果非常经典：最边缘国家和次边缘国家的系数极度显著（P值很小，说明药效极好），而次中心国家和中心国家的系数根本不显著（P值很大，说明药效不明显）

## 2.2出度中心度计算
### 2.2.1 矩阵的构建
在我们的代码中，我们计算的是无权有向图的出度中心度（Out-Degree Centrality）。这衡量了一个国家在数字服务出口网络中的“市场广度”。第一步：构建邻接矩阵（Adjacency Matrix）假设全球贸易网络中有 $N$ 个国家。令 $V_{ijt}$ 表示 $t$ 年国家 $i$ 向国家 $j$ 的数字服务出口额。为了过滤噪音并刻画核心网络，我们设定当年所有贸易流的中位数 $\tau_t = \text{median}(V_{ijt})$ 为截断阈值。我们构建一个二元指示变量 $a_{ijt}$：$$a_{ijt} = \begin{cases} 1, & \text{if } V_{ijt} \ge \tau_t \\ 0, & \text{if } V_{ijt} < \tau_t \end{cases}$$这形成了一个 $N \times N$ 的非对称 0-1 矩阵。

### 2.2.2出度中心度的计算
对于国家 $i$ 在 $t$ 年的出度（即成功建立出口连接的国家数量）为：$$K_{it}^{out} = \sum_{j \neq i} a_{ijt}$$为了消除网络规模 $N$ 随时间变化带来的不可比性，我们将其除以最大可能连接数 $(N-1)$ 进行标准化，得到出度中心度 $C_{it}$：$$C_{it} = \frac{K_{it}^{out}}{N-1}$$此时，$C_{it} \in [0, 1]$。数值越接近 1，代表该国出口市场越多元化。计算结果如下

### 2.2.3 连续敞口指数的计算
$$Exposure_{it} = \sum_{j \neq i} \left( w_{ij} \times TAPED_{ijt} \right)$$
其中，$TAPED_{ijt}$ 表示 $t$ 年国家 $i$ 和国家 $j$ 之间是否实施了数字贸易规则（或其条款深度）。$w_{ij}$ 为**静态基期贸易权重**，计算公式为：
$$w_{ij} = \frac{Trade_{ij, t_0}}{\sum_{j \neq i} Trade_{ij, t_0}}$$
本文选取数字贸易规则爆发前的 2005-2007 年作为静态基期（$t_0$）计算 $w_{ij}$。该设定不仅测度了本国面临数字贸易规则冲击的真实“敞口浓度”，还利用“基期快照”切断了“签订规则与当年贸易额激增”之间的逆向因果路径，有效缓解了内生性问题。并且我们还对敞口指数也做了一定的可视化，但是令人感到奇怪的是，我们在前二十名中并没有看到中国的影子
